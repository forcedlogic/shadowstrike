## рҹ“Ӣ Quick Start Guide

### Prerequisites
- Linux (Ubuntu, Debian, Kali Linux recommended)
- Bash shell
- Sudo access (for tool installation)
- Git (optional, for cloning)

## рҹҡҖ Installation Methods

### Method 1: Clone from GitHub (Recommended)

```bash
git clone https://github.com/forcedlogic/shadowstrike.git
cd shadowstrike
chmod +x shadowstrike.sh
./shadowstrike.sh
```

### Method 2: Download & Run Direct

```bash
wget https://raw.githubusercontent.com/forcedlogic/shadowstrike/main/shadowstrike.sh
chmod +x shadowstrike.sh
./shadowstrike.sh
```

### Method 3: Copy-Paste Script File

1. Copy the contents of `shadowstrike.sh`
2. Create a new file on your Linux machine:
```bash
nano shadowstrike.sh
```
3. Paste the script content
4. Press `Ctrl+X`, then `Y`, then `Enter`
5. Make it executable:
```bash
chmod +x shadowstrike.sh
```
6. Run it:
```bash
./shadowstrike.sh
```

## рҹҺҜ First Time Setup

When you first run ShadowStrike:

1. **Configure Your Target**
   - Press `9` for Settings
   - Press `1` for Configure Target
   - Enter your target IP, hostname, and port
   - This becomes your default for all operations

2. **Check Tools**
   - Press `9` for Settings
   - Press `2` for Check Installed Tools
   - ShadowStrike will auto-install any missing tools

## рҹ“ҡ Usage Examples

### Example 1: Port Scanning a Target

```
1. Run ShadowStrike: ./shadowstrike.sh
2. Press 1 (Reconnaissance & Scanning)
3. Press 1 (NMAP)
4. When asked "Use current target?" press Y
5. Select scan type (1 for Quick, 2 for Standard, 3 for Full)
6. View results
7. When asked "Save loot?" press Y
8. Enter loot type: "open_ports"
9. Enter loot value: "22,80,443"
```

### Example 2: Web Server Vulnerability Scan

```
1. Press 2 (Vulnerability Assessment)
2. Press 1 (Nikto)
3. When asked "Use current target?" press Y
4. Scan runs on your configured target
5. View vulnerabilities
6. Press Y to save loot
7. Enter loot type: "vulnerabilities"
8. Enter findings
```

### Example 3: Generate a Reverse Shell Payload

```
1. Press 3 (Exploitation Framework)
2. Press 2 (Reverse Shell Generator)
3. Select shell type:
   - 1 for Bash
   - 2 for Python
   - 3 for Netcat
4. Enter your attacker IP (LHOST)
5. Enter your attacker port (LPORT)
6. Copy the generated command
7. Loot is auto-saved to your inventory
```

### Example 4: View All Captured Loot

```
1. Press 5 (Loot Management)
2. Press 1 (View All Loot)
3. See all loot types and counts
4. Press a number to view details
5. Loot shows timestamps, targets, and sources
```

### Example 5: Export Loot for Report

```
1. Press 5 (Loot Management)
2. Press 3 (Export Loot)
3. Enter filename: "pentesting_report"
4. File saved as: pentesting_report_YYYYMMDD_HHMMSS.txt
5. Use for documentation and reports
```

## рҹӣ пёҸ Main Features

### рҹҺҜ Reconnaissance & Scanning
- **NMAP** - Fast, comprehensive port scanning
  - Quick scan (top 1000 ports)
  - Standard scan (common ports)
  - Full scan (all ports 1-65535)
  
- **Netdiscover** - ARP network discovery
  - Find all active hosts on a network
  - Identify MAC addresses

### рҹ”Қ Vulnerability Assessment
- **Nikto** - Web server vulnerability scanner
  - Detects dangerous files/CGIs
  - Outdated software identification
  
- **SearchSploit** - Exploit database search
  - Find public exploits for software
  - Get CVE information instantly

### рҹ’Ј Exploitation Framework
- **MSFVenom** - Payload generation
  - Windows EXE payloads
  - Linux ELF payloads
  - Android APK payloads
  
- **Reverse Shell Generator** - One-liners for quick shells
  - Bash reverse shells
  - Python reverse shells
  - Netcat reverse shells

### рҹ’° Loot Management
- **View All Loot** - See everything captured
  - Organized by type (credentials, hashes, etc.)
  - Shows count and timestamps
  
- **Manually Add Loot** - Quick entry of findings
  - Credentials, hashes, API keys, tokens, emails, domains
  - Auto-saves with target info
  
- **Export Loot** - Generate reports
  - All loot in one organized file
  - Perfect for client reports
  
- **Clear Loot** - Clean slate with confirmation
  - Safety check to prevent accidents

## рҹҡЁ Important Tips

### Best Practices
1. **Always get authorization** before testing any target
2. **Set your target first** in settings before running scans
3. **Save loot after each scan** for comprehensive documentation
4. **Export loot regularly** to avoid data loss
5. **Use custom targets** when testing multiple hosts in one session

### Auto-Install Feature
- First time running a tool? ShadowStrike will ask to install it
- Automatically updates package manager
- Requires sudo access
- Safe confirmation before installation

### Tool Requirements
- All tools are open-source
- Most available in default Linux repositories
- ShadowStrike handles installation automatically
- No dependencies between tools
